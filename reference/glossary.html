MCTS Game Engine — Glossary

Definitive reference for all technical terms used in lessons.

Core Algorithm Concepts Algorithm

Monte Carlo Tree Search (MCTS)
An anytime, best-first search algorithm that builds a partial game tree by repeatedly running four phases (Selection → Expansion → Simulation → Backpropagation). Uses random sampling to estimate move values rather than exhaustive search.
Selection
Phase 1 of the MCTS cycle. Starting at the root, the algorithm descends through the tree choosing children via a selection policy (typically UCT) until it reaches a node that hasn't been fully explored.
Expansion
Phase 2. Adds one or more new child nodes to the tree at the frontier reached during Selection.
Simulation (Rollout)
Phase 3. From the newly expanded node, play a rapid, randomized game to a terminal state to obtain a reward (win/loss/draw). This is the “Monte Carlo” sampling step.
Backpropagation
Phase 4. Propagate the simulation result back up through all nodes visited during Selection, updating their visit counts and accumulated rewards.

Selection Policy Formula

UCT (Upper Confidence Bound applied to Trees)
The standard node-selection formula: balances exploitation (choosing nodes with high average reward) with exploration (choosing nodes visited few times).
Exploitation
Preference for moves known to have high historical win rates. Pulls selection toward the best-known branch.
Exploration
Preference for less-visited moves. Helps discover unexpectedly strong branches that haven't been sampled enough.
Cp (Exploration Constant)
A tunable parameter in the UCT formula that controls the balance between exploration and exploitation. Higher values favor exploration. Often set to ~1.414 (√2) as a default per the original paper.

Node Statistics Concept

Visit Count (n(s))
How many times a particular game state (node) has been selected during the MCTS process.
Win Rate (w(s)/n(s))
The proportion of simulations starting from state s that resulted in a win for the current player.
Total Reward (w(s))
Sum of rewards accumulated from simulations passing through this node.

Game-Engine Concepts Concept

Branching Factor
The number of legal moves available at a given game state. High branching factor (e.g., ~35 in Chess, ~130 in Go) is what makes exhaustive search infeasible.
Stochastic Game
A game that includes random elements (dice rolls, shuffled decks, etc.). MCTS handles these by modelling chance nodes as additional layers in the tree.
Chance Node
A tree node representing a random event (e.g., a die roll). Instead of a player decision, it branches to all possible outcomes with their probabilities.
Determinization
A technique for handling hidden information or randomness: repeatedly fix the unknown/random elements to particular values, solve the resulting deterministic game, and aggregate across many determinizations.
Anytime Algorithm
An algorithm that can return a reasonable answer at any point and improves its answer given more computation time. MCTS is a classic anytime algorithm.

Tradeoffs Tradeoff

Depth vs. Breadth
MCTS naturally allocates more simulations to promising regions (depth) while still sampling diverse branches (breadth), unlike fixed-depth search which cuts off uniformly.
Rollout Policy Quality vs. Speed
Using a smarter (heuristic-guided) rollout produces more realistic simulations but reduces the number of simulations possible per second. A random rollout is fast but noisy.
Tree Width vs. Memory
Keeping more child nodes expands the tree's coverage but consumes more memory per simulation. MCTS typically only expands promising nodes incrementally.
