Lesson 0002: Implementing Selection in Code

Turning the UCT formula into working Python — the select_child and tree_policy functions.

1. The Shape of an MCTS Node

Every node in our search tree needs to store four things: the game state it represents, its parent link, the children it has expanded, and the statistics that UCT needs ($n$, $w$). It also needs to know which moves haven't been tried yet — because during Selection we need to know when we've hit a frontier.

Here's a minimal MCTSNode:

class MCTSNode: """A node in the MCTS search tree.""" def __init__(self, state, parent=None): self.state = state # game-specific state object self.parent = parent # parent MCTSNode (None for root) self.children = [] # list of child MCTSNode objects self._untried_moves = None # moves not yet expanded (lazy) self.visits = 0 # n(s) — total visits to this node self.wins = 0 # w(s) — total reward through this node # ── Properties ── @property def untried_moves(self): """Legal moves not yet expanded as child nodes.""" if self._untried_moves is None: self._untried_moves = self.state.legal_moves() return self._untried_moves @property def is_fully_expanded(self): """True if every legal move has been expanded.""" return len(self.untried_moves) == 0 @property def is_terminal(self): """True if this state ends the game.""" return self.state.is_terminal()
💡 Design note: _untried_moves is computed lazily from state.legal_moves() and stored so we don't recompute it. As we expand children, we remove moves from this list. The game-specific state object must define legal_moves() and is_terminal().

2. Selecting a Child with UCT

This is the core function from Lesson 0001. Given a parent node, pick the child that maximises:

$\displaystyle \text{UCT}(s,a) = \frac{w(s,a)}{n(s,a)} + C_p \sqrt{\frac{\ln N(s)}{n(s,a)}}$
import math def uct_select_child(node, Cp=1.414): """Return the child of `node` with the highest UCT score.""" best_score = -float('inf') best_child = None for child in node.children: # Unvisited children get priority (infinite UCT) if child.visits == 0: return child exploit = child.wins / child.visits explore = Cp * math.sqrt(math.log(node.visits) / child.visits) score = exploit + explore if score > best_score: best_score = score best_child = child return best_child
⚠️ Important detail: The if child.visits == 0: return child line ensures any unvisited child is chosen immediately. Because we check this before computing scores, a node with multiple unvisited children returns the first one encountered. This is fine — they'll all get visited.

3. The Tree Policy: Descending to a Frontier

Selection is not just one uct_select_child call — it's a loop that descends the tree until it finds a node that isn't fully expanded (or is terminal). This loop is called the tree policy.

def tree_policy(node, Cp=1.414): """Descend from `node` to a frontier node using UCT selection. Returns a node suitable for expansion (or a terminal node).""" while not node.is_terminal: if not node.is_fully_expanded: # Found a frontier — stop here for expansion return node else: # Fully expanded: use UCT to pick the best child node = uct_select_child(node, Cp) return node # terminal node reached
💡 Why two paths? If a node isn't fully expanded, we stop and let the Expansion phase add a new child. If it is fully expanded, we use UCT to descend deeper. This is the elegant balance: UCT guides us through known territory, and when we hit the unknown, we stop to explore.

4. Picking the Best Move After Search

After running thousands of MCTS iterations, the root node will have many children with rich statistics. To choose the final move to play, we use a different criterion: pure exploitation. The most common strategy is to pick the child with the highest visit count (robust child), not the highest win rate.

def best_move(node): """After MCTS is complete, choose the most-visited child. Returns the child node (or None if no children exist).""" if not node.children: return None return max(node.children, key=lambda c: c.visits)
Strategy Criterion When to Use
Robust child (most visits) $\arg\max n(s,a)$ Default — statistically most reliable
Max child (highest win rate) $\arg\max w(s,a) / n(s,a)$ When visit counts are similar; can overfit noise
Robust-max child Highest win rate among those with max visits Tie-breaking; combines reliability with quality

5. The Full Selection Phase, Annotated

Here's how the Selection phase fits into a single MCTS iteration:

def mcts_iteration(root, Cp=1.414): # 1. SELECTION — descend to a frontier or terminal node node = tree_policy(root, Cp) # 2. EXPANSION — if not terminal, add one new child if not node.is_terminal: node = expand(node) # expands one untried move # 3. SIMULATION — rollout from the new/terminal node reward = simulate(node) # 4. BACKPROPAGATION — update stats up the tree backpropagate(node, reward)

Notice that tree_policy may return a terminal node (e.g., checkmate on the board). In that case, expand is skipped and the simulation just returns the known result — no rollout needed.

6. Putting It All Together: A Minimal Engine

def mcts_search(root_state, iterations=1000, Cp=1.414): """Run MCTS from `root_state` and return the best action.""" root = MCTSNode(root_state) for _ in range(iterations): mcts_iteration(root, Cp) best = best_move(root) return best.state.last_action # the action that led to this child
⚠️ Key constraint: The state object must store the last_action that produced it, so we can retrieve which move to play. This is a common pattern — each node's state remembers how it was reached.

7. Check Your Understanding

✓ Quiz: Selection in Code

1. What does tree_policy return when it encounters a terminal node?

Correct! If the reached node is terminal (game over), it's returned directly. The Expansion step will skip it, and the Simulation step returns the known result.

2. Why does best_move use visit count instead of UCT score?

Correct! After enough iterations, the visit count reflects the algorithm's confidence. The child with the most visits has been explored thoroughly and converged upon. It's more robust than picking the highest win rate (which could be lucky noise from few visits).

3. If a node has 3 children all with visits > 0, what happens when Selection reaches it?

Correct! Since all children have been visited and all moves expanded, the node is fully expanded. tree_policy selects the highest-UCT child and keeps descending until it finds a frontier or terminal node.

4. What does uct_select_child do when it finds a child with visits == 0?

Correct! Unvisited children get $+\infty$ UCT, so the function short-circuits and returns the first one found. This guarantees every child is sampled at least once.

8. Summary

  • MCTSNode stores state, parent, children, $n$, $w$, and untried moves.
  • uct_select_child implements $\text{UCT} = w/n + C_p\sqrt{\ln N / n}$, with a short-circuit for unvisited children.
  • tree_policy loops down the tree until it finds a frontier (not fully expanded) or a terminal node.
  • best_move picks the most-visited child after search ends — the "robust child" strategy.
  • The state object must expose legal_moves(), is_terminal(), and last_action.
