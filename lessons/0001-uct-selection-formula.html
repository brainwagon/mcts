Lesson 0001: The UCT Selection Formula

The engine at the heart of MCTS — how to choose which branch to explore.

1. Why Selection Is the Brain of MCTS

In the MCTS cycle, Selection is where the algorithm decides which path to descend. A naive selection — always picking the move with the highest win rate — would never discover hidden strengths of unexplored branches. A random selection would be too slow to converge.

The solution is the UCT (Upper Confidence Bound applied to Trees) formula. It treats each move from a game state as a multi-armed bandit problem: you have several slot machines (moves), each paying out wins at some unknown rate. UCT tells you which machine to pull next.

2. The Three Functions You Need to Know

Before we unpack the formula, let's name every piece. UCT operates on a tree where each node represents a game state, and each edge (parent $→$ child) represents taking an action $a$ from state $s$.

Symbol Name What It Means
$N(s)$ Parent visit count Total times the parent state $s$ has been visited across all MCTS iterations so far.
$n(s,a)$ Child visit count Number of times this specific move $a$ was chosen from state $s$. Starts at 0, increments each time Selection descends into it.
$w(s,a)$ Total reward Sum of all rewards (e.g., wins $=1$, losses $=0$) obtained from simulations that passed through this child. $w(s,a)\,/\,n(s,a)$ is the empirical win rate.
$C_p$ Exploration constant A tunable parameter (default $\approx \sqrt{2}$) that controls how much weight to give exploration vs. exploitation.

So the UCT formula takes these four values and combines them into a single score. The child with the highest UCT score gets selected.

3. The UCT Formula (Anatomy)

$\displaystyle \text{UCT}(s,a) = \underbrace{\frac{w(s,a)}{n(s,a)}}_{\text{exploitation}} \;+\; \underbrace{C_p \sqrt{\frac{\ln N(s)}{n(s,a)}}}_{\text{exploration}}$
Term Meaning Role
$w(s,a) / n(s,a)$ Win rate of action $a$ from state $s$ Exploitation — favour moves known to win
$C_p \sqrt{ \ln N(s) / n(s,a) }$ Uncertainty bonus — shrinks as we visit $a$ more Exploration — favour moves tried fewer times
$C_p$ Exploration constant (tunable, default $\approx \sqrt{2}$) Balance — how much weight to give exploration

When $n(s,a) = 0$ (the move has never been tried), the formula returns $+\infty$ — it always selects an unvisited child. This ensures every legal move is sampled at least once before exploitation dominates.

💡 Key Insight: The exploration term $\sqrt{\ln N(s) / n(s,a)}$ uses the parent's total visits $N(s)$ in the numerator, not the child's. This means a move that looks average but has been visited far fewer times than its siblings gets a large exploration bonus.

5. Worked Example (By Hand)

Suppose we're at a root with $N(s) = 100$ total visits, comparing two child moves:

  • Move X: $n=20,\; w=15$ (75% win rate)
  • Move Y: $n=5,\; w=3$ (60% win rate)

With $C_p = 1.41\;(\approx\sqrt{2})$:

Move Exploitation $w/n$ Exploration $C_p\sqrt{\ln N / n}$ UCT
X $15/20 = 0.750$ $1.41 \times \sqrt{\ln 100 / 20} = 0.484$ $1.234$
Y $3/5 = 0.600$ $1.41 \times \sqrt{\ln 100 / 5} = 0.968$ $1.568$

Result: Move Y is selected even though its win rate is lower. Why? Because it has been visited only 5 times vs. 20 for X — the algorithm is uncertain about Y and wants to gather more data.

🧠 Why this matters for engine design: In high-branching games like Risk (where a single territory can be attacked from multiple fronts), this behaviour lets the engine try new attack patterns early — even if they look weak — while quickly converging on the statistically strongest lines.

6. Tuning $C_p$ for Your Game

$C_p$ Value Behaviour Best For
$0$ Pure exploitation (greedy) Late game with few moves left; known-optimal endgames
$0.5$ – $1.0$ Light exploration Deterministic games with simple rules (Tic-Tac-Toe)
$1.414\;(\sqrt{2})$ "Goldilocks" default Most board games; proven optimal for bandit problems
$2.0$ – $3.0$ Heavy exploration Stochastic games (dice/randomness) where the same move can have varying outcomes
$> 3.0$ Near-random selection Early in training; discovering initial strategies

7. Check Your Understanding

✓ Quiz: UCT Fundamentals

1. If a child node has $n(s,a) = 0$ (never visited), what UCT value does it receive?

Correct! An unvisited child always gets priority — $w/n$ diverges to $+\infty$, guaranteeing every legal move is sampled at least once.

2. What happens to the exploration term $\sqrt{\ln N(s) / n(s,a)}$ as $n(s,a)$ increases?

Correct! As we visit a child more times, the uncertainty about its true win rate shrinks, so the exploration bonus diminishes. The algorithm shifts from exploration to exploitation.

3. You're designing an engine for Risk (dice-based combat). Should $C_p$ be higher or lower than the default $\sqrt{2}$?

Correct! Risk has randomness (dice rolls), so the same move can produce different outcomes. More exploration (higher $C_p$) helps the engine gather enough samples to see past the noise.

8. Summary

  • $\displaystyle \text{UCT} = \underbrace{\frac{w}{n}}_{\text{exploitation}} \;+\; C_p \underbrace{\sqrt{\frac{\ln N}{n}}}_{\text{exploration}}$
  • Unvisited children are always selected first (infinite UCT).
  • Higher $C_p$ = more exploration; lower = more exploitation.
  • Stochastic games (dice, cards) benefit from higher $C_p$.
  • Selection is called repeatedly as the first phase of every MCTS iteration.
