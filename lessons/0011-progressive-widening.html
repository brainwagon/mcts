Lesson 0011: Progressive Widening

Solving the Curse of Dimensionality in massively stochastic environments.

1. The Curse of Wide Chance Nodes

In Lesson 0010, we explicitly modeled Chance Nodes to properly evaluate expected value in a stochastic game. However, Yahtzee dice rolls have up to 252 possible outcomes. If a Chance Node blindly explores every unique dice roll it encounters, its visits get spread disastrously thin across hundreds of children.

This means if an action gets visited 15 times during MCTS, it might spawn 15 distinct children. Each child has only visits = 1, resulting in massive variance and noise. The algorithm can never confidently evaluate the action because it never samples the same outcome twice to build statistical significance.

2. The Theory: Progressive Widening

Progressive Widening (PW) is an elegant mathematical solution to this problem. Instead of blindly letting a Chance Node spawn infinite children, we artificially restrict its branching factor based on how many times the node has been visited.

The standard formula limits the maximum number of allowed children (C) to:

C = ⌈k × Nα⌉

Where N is the number of visits to the node, and k and α are tunable parameters (commonly k = 2.0, α = 0.5).

As the node receives more visits, we progressively "widen" it by allowing it to explore more unique chance outcomes. But initially, we force it to repeatedly test just a handful of outcomes so it can achieve statistical depth.

3. The Implementation

In our ChanceNode, we added a natural_counts dictionary to track how often the environment naturally produces specific outcomes.

class ChanceNode: def __init__(self, action: Tuple[str, Any], parent: 'DecisionNode'): self.action = action # ... self.children: Dict[YahtzeeState, 'DecisionNode'] = {} self.natural_counts: Dict[YahtzeeState, int] = {}

During the Tree Policy traversal, we calculate the allowed width. If the node is full, we block the sampling of a new state and instead force the algorithm to traverse one of the existing children. Crucially, we use random.choices weighted by natural_counts to perfectly preserve the true probability distribution of the dice!

max_children = max(1, math.ceil(k * (best_chance_node.visits ** alpha))) if len(best_chance_node.children) < max_children: # Allowed to widen! Sample a new outcome from the environment. next_state = node.state.apply(best_action) if next_state not in best_chance_node.children: # Create a new branch child_decision = DecisionNode(next_state, parent=best_chance_node) best_chance_node.children[next_state] = child_decision best_chance_node.natural_counts[next_state] = 1 else: best_chance_node.natural_counts[next_state] += 1 else: # Restricted! Must sample from already discovered outcomes based on natural frequency. states = list(best_chance_node.children.keys()) weights = [best_chance_node.natural_counts[s] for s in states] next_state = random.choices(states, weights=weights, k=1)[0]

4. The Final 50,000 Iteration Benchmark

To truly test if Progressive Widening solved our Yahtzee engine, we ran a massive 50,000-iteration benchmark (with Cp = 40.0). Here is how it compares to our previous attempts:

  • Open-Loop MCTS (500 iters): ~156.5 pts
  • ExpectiMax MCTS (500 iters): ~152.2 pts
  • ExpectiMax MCTS (50,000 iters): ~184.2 pts
  • 0-ply Search (Pure Heuristic): ~191.3 pts
  • ExpectiMax + PW MCTS (500 iters): ~162.0 pts
  • ExpectiMax + PW MCTS (50,000 iters): ~190.5 pts
The Grand Conclusion: Even with Progressive Widening restricting the chance nodes, and even after throwing massive computational power (50,000 iterations) at the problem, the MCTS only managed to climb to 190.5 points, mathematically tying the raw 0-ply heuristic (191.3 points).

This is the ultimate punchline of our AI experiment. It perfectly illustrates the limitations of Monte Carlo Tree Search. MCTS is a phenomenal algorithm for games with massive state spaces and deep, deterministic tactics (like Chess or Go). However, in a heavily stochastic game like Yahtzee, MCTS acts as a statistical sledgehammer. To truly beat the heuristic, the algorithm would require millions of iterations to converge on the exact probabilities.

For games like Yahtzee or Blackjack, the state space is actually surprisingly small. The computationally correct approach is never MCTS; it is Value Iteration / Dynamic Programming, which simply calculates the perfect expected value from the bottom up.
