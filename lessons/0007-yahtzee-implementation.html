Lesson 0007: MCTS Plays Yahtzee

A complete, runnable MCTS engine for Yahtzee — demonstrating stochastic actions, single-player adaptation, and heuristic rollouts in one compact implementation.

1. Why Yahtzee?

Yahtzee is an ideal test case for bringing together every concept from this course:

  • High branching factor — up to 32 keep-masks plus 13 scoring categories
  • Randomness — dice rolls are stochastic transitions
  • Single-player optimization — no opponent, so no sign alternation in backpropagation
  • Deep strategy — when to score vs. re-roll, which categories to fill first, Yahtzee bonus chasing

2. Key Design Decisions

2.1 MCTS Action Space

Two kinds of actions (the same as Lesson 0005's stochastic game pattern):

  • KEEP(mask) — keep a subset of dice and re-roll the rest. This is a stochastic transition (dice outcomes are random).
  • SCORE(category) — score the current dice in a category. This is a deterministic scoring action, but transitioning to the next turn rolls new random dice.

2.2 Open Loop MCTS for Chance Nodes

Because Yahtzee involves random dice rolls, storing specific deterministic game states inside the tree nodes causes "optimism" bugs (the AI might assume it will always get the lucky dice it rolled when first expanding a node). Instead, we use Open Loop MCTS: the tree nodes only represent action sequences, and the actual game state is dynamically re-sampled during each tree traversal by applying the actions.

2.3 Single-Player Reward

Yahtzee has no opponent. The reward is the final total score at the terminal state (all 13 categories filled). Backpropagation does not alternate sign — every node along the path adds the same positive reward. UCT selects children that maximize the average final score from that state.

2.4 Proper $C_p$ Scaling

Yahtzee scores are typically around 100–300 points. The standard $C_p$ of 2.5 assumes normalized rewards between [0, 1]. For unnormalized scores, we scale $C_p$ to 10.0 to properly balance exploration and exploitation without acting purely greedily or purely randomly.

2.5 Rollout Policy

Pure random rollouts would produce terrible Yahtzee play (randomly re-rolling random dice). Instead, we use a simple heuristic:

  1. If any available category scores ≥ 20 points from the current dice, score it immediately.
  2. Otherwise, keep dice that show the most frequent value (building toward N-of-a-kind) and re-roll the rest.

3. The Code

The complete implementation is in yahtzee_mcts.py, a single ~250-line Python file. Here are the key sections:

3.1 Scoring

# Category scoring for all 13 Yahtzee categories (0–12). # Returns the points earned (0 for invalid/unfilled). def score_dice(dice, category): counts = [dice.count(v) for v in range(1, 7)] if category < 6: # Upper section: Ones through Sixes return (category + 1) * counts[category] if category == 6: # Three of a Kind return sum(dice) if max(counts) >= 3 else 0 if category == 7: # Four of a Kind return sum(dice) if max(counts) >= 4 else 0 if category == 8: # Full House (3+2) return 25 if sorted(counts) == [0,0,0,0,2,3] else 0 if category == 9: # Small Straight (4 consecutive) dice_set = set(dice) straights = [{1,2,3,4}, {2,3,4,5}, {3,4,5,6}] return 30 if any(s.issubset(dice_set) for s in straights) else 0 if category == 10: # Large Straight (5 consecutive) return 40 if sorted(dice) in [[1,2,3,4,5], [2,3,4,5,6]] else 0 if category == 11: # Yahtzee (5 of a kind) return 50 if max(counts) == 5 else 0 if category == 12: # Chance return sum(dice)

3.2 State and Stochastic Transitions

class YahtzeeState: def __init__(self, dice, rolls_left, scores): self.dice = tuple(sorted(dice)) # canonical order self.rolls_left = rolls_left self.scores = list(scores) # -1 = unfilled, 13 categories def legal_moves(self): moves = [] for cat in range(13): if self.scores[cat] < 0: moves.append(('SCORE', cat)) if self.rolls_left > 0 and not self.is_terminal(): for mask in range(1 << 5): moves.append(('KEEP', mask)) return moves def apply(self, move): kind, arg = move if kind == 'KEEP': # Stochastic: randomly fill non-kept positions new_dice = [] for i in range(5): if arg & (1 << i): new_dice.append(self.dice[i]) else: new_dice.append(random.randint(1, 6)) return YahtzeeState(new_dice, self.rolls_left - 1, self.scores) else: # SCORE cat = arg raw = score_dice(self.dice, cat) bonus = 0 # Yahtzee bonus: +100 for extra Yahtzees after scoring cat 11 if len(set(self.dice)) == 1 and self.scores[11] >= 0 and cat != 11: bonus = 100 new_scores = list(self.scores) new_scores[cat] = raw if all(s >= 0 for s in new_scores): # Terminal state return YahtzeeState(self.dice, 0, new_scores) # Next turn: fresh roll fresh = tuple(sorted(random.randint(1, 6) for _ in range(5))) return YahtzeeState(fresh, 2, new_scores)

3.3 Open Loop MCTS Engine

The key adaptation: MCTSNode no longer stores a YahtzeeState. It only stores the action that led to it. The state is re-sampled dynamically during the tree policy traversal.

class MCTSNode: def __init__(self, parent=None, action=None): self.parent = parent self.action = action self.children = [] self.visits = 0 self.total_reward = 0.0 self.untried_moves = None

3.4 Heuristic Rollout

def simulate(node): state = node.state.clone() while not state.is_terminal(): moves = state.legal_moves() score_moves = [m for m in moves if m[0] == 'SCORE'] keep_moves = [m for m in moves if m[0] == 'KEEP'] # Heuristic: if any category scores ≥20, take it. best_now = max((score_dice(state.dice, cat), cat) for cat in range(13) if state.scores[cat] < 0) if best_now[0] >= 20 or not keep_moves: state = state.apply(('SCORE', best_now[1])) else: # Keep the most frequent die value; re-roll rest most_common = max(range(1, 7), key=lambda v: state.dice.count(v)) mask = 0 for i, d in enumerate(state.dice): if d == most_common: mask |= (1 << i) state = state.apply(('KEEP', mask)) return float(state.current_total())

4. Running It

# Play interactively (human decides each move, AI helps via MCTS): $ python yahtzee_mcts.py # Watch the AI play 100 games automatically: $ python yahtzee_mcts.py --simulate 100

5. What to Expect

With 500 iterations per decision, the AI scores around 150–200 points on average. A perfect player scores ~250–300. The gap comes from:

  • Limited iterations — more iterations = better play
  • Simple rollout heuristic — the "keep most common" heuristic is naive; smarter heuristics improve play
  • No explicit chance nodes — open loop MCTS elegantly averages out the randomness during tree traversal without bloating the tree.
🧠 Experiment ideas:
  • Increase iterations from 500 to 5000 — how much does the score improve?
  • Change $C_p$ — try 1.0, 2.5, 5.0 and compare average scores
  • Improve the rollout heuristic — e.g., also keep pairs for Full House potential

6. Check Your Understanding

✓ Quiz: Yahtzee MCTS

1. Why does this implementation use Open Loop MCTS (nodes don't store states)?

Correct! By not caching states in nodes, we force the engine to re-roll the dice on every traversal, which naturally averages out the randomness of actions like KEEP over many iterations.

2. How does backpropagation differ for single-player Yahtzee vs. two-player Chess?

Correct! Since there's no opponent, the reward doesn't flip sign. Each node accumulates the total final score, and UCT maximizes average score.

3. Why did we scale $C_p$ to 10.0 instead of using the standard 2.5?

Correct! The exploration term must be on a comparable scale to the exploitation term (the average score). For rewards in the hundreds, $C_p$ must be scaled up appropriately.

4. What score does the Yahtzee bonus add?

Correct! After marking the Yahtzee box (50 pts), each subsequent Yahtzee rolled earns a 100-point bonus.

7. Summary

  • Yahtzee brings together stochastic transitions, high branching factor, and single-player optimization.
  • Open Loop MCTS avoids optimism in chance outcomes by re-sampling states dynamically instead of storing them in nodes.
  • Scaled $C_p$ balances exploration with the unnormalized high point scores of Yahtzee.
  • Backpropagation has no sign flip (single-player reward).
  • A heuristic rollout (score ≥20 or keep most frequent) provides a baseline for the tree.
  • The complete implementation is ~250 lines of Python — run it with python yahtzee_mcts.py.
